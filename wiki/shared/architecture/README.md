# Architecture

Drop architecture diagrams and high-level design docs here: cross-app data flows,
infrastructure diagrams, system maps, and anything describing how the pieces fit
together across more than one app.

- Keep app-internal design in that app's own section. This folder is for the
  **system-level** view.
- Diagrams can be images, `.canvas` files, Mermaid in markdown, or exported PDFs —
  whatever communicates best.
- When a diagram reflects a decision, link it to the relevant
  [[../decisions/README|ADR]]; when it documents an integration, link the matching
  [[../api-contracts/README|API contract]].
